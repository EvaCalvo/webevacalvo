FASTERCOOK

Fastercook es una aplicación para móvil que permite tener menús semanales equilibrados y cuya elaboración no supera los 30 minutos.

El principal problema detectado y que llevó al desarrollo de esta app, fue el poco tiempo del que dispone la mayoría de los usuarios tanto para elaborar menús equilibrados como para cocinarlos.

Las fases por las que pasé para desarrollar esta app fueron las siguientes:



EMPATIZAR:


Para ello debía ponerme en el lugar de los usuarios; conocer cómo era su día a día y sus preocupaciones a la hora de llevar una dieta equilibrada, tanto para él como para los miembros de su familia.

Uno de los datos obtenidos más comunes, fue la preocupación por una alimentación sana que evitara en la medida de lo posible tener que cocinar alimentos procesados o congelados. El tiempo del que disponen para dedicar a las tareas de planificación semanal de menús, elaboración de platos y creación de listas de la compra, les impiden llevar una dieta tan equilibrada como les gustaría; lo cual les inquieta, no sólo por ellos mismos sino también por el resto de los componentes de la unidad familiar.

Muchos de nuestros usuarios buscan recetas en internet o se apoyan en libros de cocina pero, la mayoría de los platos propuestos, suponen un tiempo de elaboración demasiado elevado.

Recabé toda esta información a partir de procesos como: Blueprint, Problem Statement, Competitive Analysis, Lean Survey Canvas, Interviews.



DEFINIR:


Analizando los datos obtenidos, pude definir un poco más cuál era el problema real de nuestros usuarios, así como las características del mismo.

Me ayudó emplear procesos como: Lean UX Canvas, User Persona, Empathy Map, User Journey, Affinity Diagram.

La falta de tiempo en todos los casos es el factor determinante para que no puedan llevar una dieta tan equilibrada como les gustaría.



IDEAR:


Detectado el problema principal, había que pensar soluciones para facilitar a los usuarios alcanzar su objetivo de comer equilibradamente.

Dichas soluciones vendrían dadas a partir de la aplicación de diferentes procesos como: Brainstorming, Mindmap, User Stories.

Había que buscar funcionalidades concretas para pudieran ahorrar tiempo en las tareas que componen la alimentación. Necesitaba ofrecerles una aplicación que les permitiera generar de manera automática una serie de menús semanales adaptados a las necesidades de cada grupo de usuarios, con recetas que no superen los 30 minutos de elaboración y puedan ser visualizadas para ayudarles en la preparación, así como la posibilidad de cambiar algunos de los platos generados para adaptarlos a los gustos personales de cada usuario. Además sería conveniente crear listas de la compra a partir de dichos menús, para evitar que pierdan más tiempo en mirar los productos necesarios para llevar a cabo dichas recetas.

Una vez definida esta parte, tenía que ubicar de manera correcta y clara las funcionalidades dentro de la app. Crear un correcto flujo de usuario que permita acceder a todas las secciones de manera natural, sin hacerles pensar en cuál es el siguiente paso o cómo pueden llegar al apartado que desean.

Todo ello se determinó con procesos como: Moscow, Sitemap, Card Sorting, Userflow, Crazy 8.



PROTOTIPAR Y VALIDAR:


La validación se hace de manera constante en cada fase, para ir aprendiendo del usuario y poder refinar el modelo y las funcionalidades encontradas.

Una vez llegados a este punto, era necesario realizar un prototipo en baja fidelidad para que nuestro usuario pudiera “navegar” por la futura aplicación pudiera detectar problemas que impidieran un flujo correcto.

Validada esta parte, realicé otro prototipo, esta vez algo más detallado y realizado con herramientas como Sketch para continuar con las pruebas. Aún no tenía color, ya que no quería distraer al usuario con detalles que en ese momento no aportaban nada a esta fase.

Una vez validado con los usuarios este segundo prototipo, pasé a la parte de visual.

Los elementos visuales que darían vida a la app, tenían que surgir a partir de estudios de: Brand Attributes, Moodboard, Picking Color, Pairing Fonts, Stile Tiles, Alignment, Spacing, Consistency, Shadows…

La app tenía que reflejar salud, rapidez y eficacia. Para ello elegí imágenes, tipografías y formas que sugirieran todo ello y a partir de ahí comencé a darle forma y color a Fatercook.

Como next steps se desarrollará la funcionalidad de listas de la compra y la posibilidad de que éstas sean exportadas a las plataformas de supermercados online que existen hoy en día; consiguiendo así mayor agilidad en las tareas relativas a la alimentación y ahorrando tiempo a los usuarios.

Fastercook fue el primer proyecto realizado de forma individual dentro del bootcamp de Diseño UX / UI de Ironhack, y contamos únicamente con dos semanas para su elaboración.
